## 4. 启动测试

### 4.1 第一轮干净构建（失败，已修复）

最初执行 `make down && make up` 时，构建失败。根因为 v2.0 上游重新引入了 Docker build 阶段的 `HTTP_PROXY`/`HTTPS_PROXY` 参数，而宿主机 `host.docker.internal:6152` 代理未运行，导致 `apt-get update` 无法获取系统包。

### 4.2 第二轮干净构建（成功）

移除 build-time 代理、升级到 Python 3.13、修正 Makefile volume/image 名后，重新执行 `make down && make up`，构建与启动均成功。
